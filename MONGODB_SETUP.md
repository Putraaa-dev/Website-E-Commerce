# MongoDB Setup & Product CRUD Guide

## MongoDB Configuration

### Current Setup
- **Database**: `kodemurah` (MongoDB)
- **Connection String**: `mongodb://127.0.0.1:27017/kodemurah`
- **Collections**: `products`, `admins`

## How to Install MongoDB

### Windows

#### Option 1: MongoDB Community Edition (Recommended)
1. Download from: https://www.mongodb.com/try/download/community
2. Choose Windows, then download the `.msi` installer
3. Run the installer with default settings
4. MongoDB will be installed and running as a Windows service

#### Option 2: Using Chocolatey
```powershell
choco install mongodb
```

#### Option 3: Using Docker
```bash
docker run -d -p 27017:27017 --name mongodb mongo:latest
```

### Verify MongoDB is Running
```powershell
# Check if mongod process is running
Get-Process mongod -ErrorAction SilentlyContinue

# Or test connection
mongo --version
```

## Backend Configuration

The backend (`backend/.env`) is already configured:
```
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/kodemurah
```

If you use a different MongoDB instance, update `MONGO_URI`:
```
MONGO_URI=mongodb://username:password@host:port/dbname
```

## Product CRUD Operations

### 1. Create Product (Add Product)

**UI Steps:**
1. Go to: http://localhost:5173/admin
2. Login with: `admin@example.com`
3. Click "Products" in sidebar
4. Click "Add Product" button
5. Fill in the form:
   - Name (required)
   - Price (number)
   - Category
   - Description
   - Image URL
   - Downloads
   - Language
6. Click "Save Product" button

**What Happens:**
- Form data sent to: `POST /api/products`
- Data saved to MongoDB `products` collection
- Product ID auto-generated by MongoDB
- Product appears in table immediately
- Success toast notification shown

**API Request Example:**
```bash
curl -X POST http://localhost:5000/api/products \
  -H "Content-Type: application/json" \
  -d '{
    "name": "E-Commerce Site",
    "price": 150000,
    "category": "Web",
    "description": "Full stack e-commerce",
    "image": "https://example.com/image.jpg",
    "downloads": 0,
    "techSpecs": {
      "language": "React"
    }
  }'
```

### 2. Read Products (View List)

**UI Steps:**
1. Products appear automatically on `/admin/products`
2. Shows all products from MongoDB
3. Search by name or category

**API Request:**
```bash
curl http://localhost:5000/api/products
```

**Response:**
```json
[
  {
    "_id": "507f1f77bcf86cd799439011",
    "name": "E-Commerce Site",
    "price": 150000,
    "category": "Web",
    "description": "Full stack e-commerce",
    "image": "https://example.com/image.jpg",
    "downloads": 0,
    "createdAt": "2026-02-05T10:30:00.000Z",
    "updatedAt": "2026-02-05T10:30:00.000Z"
  }
]
```

### 3. Update Product (Edit)

**UI Steps:**
1. Click Edit button (pencil icon) on a product row
2. Form opens with current data
3. Modify fields as needed
4. Click "Save Product" button

**What Happens:**
- Data sent to: `PUT /api/products/{id}`
- MongoDB document updated
- Table refreshes with new data
- Success notification shown

**API Request:**
```bash
curl -X PUT http://localhost:5000/api/products/507f1f77bcf86cd799439011 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Name",
    "price": 200000
  }'
```

### 4. Delete Product (Remove)

**UI Steps:**
1. Click Delete button (trash icon) on a product row
2. Product removed immediately
3. Success notification shown

**What Happens:**
- Request sent to: `DELETE /api/products/{id}`
- Document removed from MongoDB
- Table updates automatically

**API Request:**
```bash
curl -X DELETE http://localhost:5000/api/products/507f1f77bcf86cd799439011
```

## Product Model (MongoDB Schema)

```javascript
{
  _id: ObjectId,           // Auto-generated MongoDB ID
  name: String,            // Product name (required)
  price: Number,           // Price in IDR
  category: String,        // Category (Web, Android, etc)
  description: String,     // Product description
  image: String,           // Image URL
  downloads: Number,       // Download count
  techSpecs: {
    language: String       // Programming language
  },
  createdAt: Date,         // Auto-created by MongoDB
  updatedAt: Date          // Auto-updated by MongoDB
}
```

## Troubleshooting

### "MongoDB Connected" not showing
**Problem:** Backend can't connect to MongoDB
**Solution:**
1. Check if MongoDB is running: `Get-Process mongod`
2. Start MongoDB service (if installed):
   ```powershell
   # Windows Service
   net start MongoDB
   ```
3. Check MONGO_URI in `backend/.env`
4. Restart backend server

### "Cannot POST /api/products"
**Problem:** Product creation fails
**Solution:**
1. Check browser console for error message
2. Check backend terminal for error logs
3. Verify all required fields are filled
4. Check MongoDB connection status

### Products not saving to database
**Problem:** Products appear in UI but not in MongoDB
**Solution:**
1. Check backend logs for error messages
2. Look for MongoDB connection errors
3. Verify database name in MONGO_URI
4. Ensure MongoDB service is running

### Port 5000 already in use
```powershell
# Find and kill process on port 5000
netstat -ano | findstr :5000
taskkill /PID <PID> /F
```

## Database Inspection

### Using MongoDB Compass (GUI)
1. Download: https://www.mongodb.com/products/compass
2. Connect to: `mongodb://localhost:27017`
3. View `kodemurah` database
4. Browse `products` collection

### Using Mongo Shell (CLI)
```javascript
// Connect to database
mongo

// Select database
use kodemurah

// View all products
db.products.find().pretty()

// View specific product
db.products.findOne({ _id: ObjectId("...") })

// Count products
db.products.count()

// Delete product
db.products.deleteOne({ _id: ObjectId("...") })
```
